About

This repository packages Cypress tests inside a Docker container, allowing you to run full end-to-end suites without installing Node, Cypress, or any browsers locally.

Running Cypress in Docker ensures tests are:

  • Fully reproducible — same environment for all developers and CI
  • Isolated — no dependency conflicts
  • Easy to run — just one Docker command

Running Tests

Method 1 — Using Docker Compose (recommended) 

docker-compose up
Method 2 — Manually build & run Docker image 

docker build -t cypress_docker .
docker run --rm cypress_docker

Docker Setup

The environment is composed of:

  • Dockerfile — sets up Node, Cypress, required dependencies
  • docker-compose.yml — runs the tests inside the container

This makes it extremely easy to integrate Cypress with CI tools like GitHub Actions, GitLab CI, Jenkins, or CircleCI.

Notes

  • You can add environment variables in docker-compose.yml.
  • Test files live inside cypress/e2e/.
  • Update cypress.config.js with your project URL and test settings.
